# 第一章 C\+\+基础 多态与虚函数

## 1\.多态是什么

多态（Polymorphism）就是“同一接口，多种实现”。在C\+\+里，它主要指通过基类的指针或引用调用函数，实际执行的是派生类自己定义的版本。

## 2\.多态的实现原理\-\>虚函数

```C++
#include<iostream>
using namespace std;
class Animal
{
public:
    void dospeak()
    {
        cout<<"Animal can speak"<<endl;
    }
    
};

class Dog:public Animal
{
public:
    void dospeak()
    {
        cout<<"Dog can bark"<<endl;
    }
};
void speak(Animal &a)
{
    a.dospeak();
}
int main()
{
    Dog d;
    speak(d);
    return 0;
}

```

```Plain Text
Animal can speak
```

```C++
#include<iostream>
using namespace std;
class Animal
{
public:
    //使用到虚函数
    /*实现多态的条件：
        1.将父类进行virtual虚函数的关键字进行修饰。
        2.在继承的子类中对虚函数进行重写
    */
    virtual void dospeak()
    {
        cout<<"Animal can speak"<<endl;
    }
    
};

class Dog:public Animal
{
public:
    void dospeak()
    {
        cout<<"Dog can bark"<<endl;
    }
};
class Cat:public Animal
{
public:
    void dospeak()
    {
        cout<<"Cat can meow"<<endl;
    }
};
void speak(Animal &a)
{
    a.dospeak();
}
int main()
{
    Dog d;
    Cat c;
    speak(d);
    speak(c);   
    return 0;
}

```

```Plain Text
Cat can meow
```

## 3\.多态的原理剖析

**多态的实现机制**：

- 虚函数表（V\-Table）：C\+\+运行时使用虚函数表来实现多态。每个包含虚函数的类都有一个虚函数表，表中存储了指向类中所有虚函数的指针。

- 虚函数指针（V\-Ptr）：对象中包含一个指向该类虚函数表的指针。

## 4\.纯虚函数与抽象类

在多态中,通常父类中虚函数的实现是毫无意义的,主要都是调用子类重写的内容。因此可以将虚函数改为纯虚函数



纯虚函数语法:

virtual 返回值类型 函数名 \(参数列表\)=0;当类中有了纯虚函数,这个类也称为抽象类

抽象类特点:

- 无法实例化对象

- 子类必须重写抽象类中的纯虚函数,否则也属于抽象类

## 5\.override 与 final 关键字

`override` —— 显式声明覆盖意图

在派生类中标记一个虚函数，告知编译器该函数意图覆盖基类中的虚函数。若实际没有覆盖任何基类虚函数（如函数签名不匹配、基类无此虚函数、基类函数为 `final` 等），编译器会报错，从而避免因拼写错误或参数类型错误导致的意外重载。

```C++
class Base {
    virtual void foo(int);
    virtual void bar() const;
};

class Derived : public Base {
    void foo(double) override;   // 错误：基类无 foo(double) —— 签名不匹配
    void bar() override;         // 错误：基类 bar 是 const，派生类缺少 const
    void foo(int) override;      // 正确
};
```

`final` —— 终止继承或覆盖

- 用于虚函数：阻止派生类进一步覆盖该函数。

```C++
class Base {
    virtual void f() final;
};
class Derived : public Base {
    void f() override;  // 错误：Base::f 已声明为 final
};
```

用于类：阻止其他类继承该类。

```C++
class FinalClass final {};
class Sub : public FinalClass {};  // 错误：不能继承 final 类
```

## 2\.用户权限与管理

### 2\.1用户管理

MySQL用户可以分为**普通用户**和**root用户**。root用户是超级管理员,拥有所有权限,包括创建用户、删除用户、修改用户信息等管理权限。普通用户只能拥有被授予的各种权限。

MySQL提供了很多语句用来管理用户账号。执行语句可以用来管理包括登录和退出MySQL服务器、创建用户、删除用户、密码管理和权限管理等内容。MySQL数据库的安全性需要通过账户管理来保证。

#### 2\.1\.1登录MySQL服务器

以前我们都是使用命令mysql \-u root \-p的命令来进行数据库登录,这种登录的方式很多参数都是默认值。我们也可以使用更为详细的命令来进行数据库的登录。

```Shell
#此方法适用于只对数据库做一次操作并连接数据库。
mysql -uroot -p -hlocalhost -P3306 mysql -e "SQL语句"
#一般命令：
mysql -h hostname|hostIP -P port -u username -p DatabaseName -e "SQL语句"
```

> 下面详细介绍命令中的参数:
> 
> - \-h参数后面接主机名或者主机IP,hostname为主机,hostIP为主机IP。
> 
> - \- P参数后面接MySQL服务的端口,通过该参数连接到指定的端口。MySQL服务的默认端口是3306,不使用该参数时自动连接到3306端口,port为连接的端口号。
> 
> - \- u参数 后面接用户名,username为用户名。
> 
> - \- p参数 会提示输入密码。
> 
> - DatabaseName参数指明登录到哪一个数据库中。如果没有该参数,就会直接登录到MySQL数据库中,然后可以使用USE命令来选择数据库。
> 
> - \- e参数 后面可以直接加SQL语句。登录MySQL服务器以后即可执行这个SQL语句,然后退出MySQL服务器。
> 
>

### 2\.2用户密码管理

#### 2\.2\.1设置当前用户的密码

root用户拥有很高的权限,因此必须保证root用于的密码安全。root用户可以通过多种方式来修改自己的密码。使用ALTER USER语句修改用户密码是MySQL官方推荐的方式。此外还可以通过SET语句来修改自己的密码。由于MySQL8已经移除了PASSWORD\(\)函数,因此不再使用update语句直接操作用户表修改密码。

这里介绍官方推荐的写法:

1. 使用ALTER USER的方式来修改当前用户密码

用户可以通过ALTER语句来修改自身的密码,如下语句代表当前用户登录的密码,基本语法如下:

```SQL
ALTER USER USER() IDENTIFIED BY '新密码';
```

示例:将当前root用户密码修改成'Hello\_1234':

```SQL
ALTER USER USER() IDENTIFIED BY 'Hello_1234'
```

2. 使用SET语句来修改当前用户密码

使用root用户登录MySQL之后,可以使用set语句来进行密码的修改。具体写法如下:

```SQL
SET PASSWORD='新密码’;
```

### 2\.3用户权限管理

#### 2\.3\.1MySQL权限分类

> \(1\)CREATE和DROP权限,可以创建新的数据库和表,或删除\(移掉\)已有的数据库和表。如果将MySQL数据库中的DROP权限授予某用户,用户就可以删除MySQL访问权限保存的数据库。
> 
> \(2\)SELECT、INSERT、UPDATE和DELETE权限 允许在一个数据库现有的表上实施操作。
> 
> \(3\)SELECT权限 只有在它们真正从一个表中检索行时才被用到。
> 
> \(4\)INDEX权限允许创建或删除索引,INDEX适用于已有的表。如果具有某个表的CREATE权限,就可以在CREATE TABLE语句中包括索引定义。
> 
> \(5\) ALTER权限 可以使用ALTER TABLE来更改表的结构和重新命名表。
> 
> \(6\)CREATE ROUTINE权限 用来创建保存的程序\(函数和程序\),ALTER ROUTINE权限用来更改和删除保存的程序,EXECUTE权限用来执行保存的程序。
> 
> \(7\)GRANT权限 允许授权给其他用户,可用于数据库、表和保存的程序。
> 
> \(8\)FILE权限 使用户可以使用LOAD DATA INFILE和SELECT \.\.\. INT OUTFILE语句读或写服务器上的文件,任何被授予FILE权限
> 
> 的用户都能读或写MySQL服务器上的任何文件\(说明用户可以读任何数据库目录下的文件,因为服务器可以访问这些文件\)。
> 
>


该权限命令的执行如果发现没有该用户,则会直接新建一个用户。

#### 2\.4\.3删除角色

`DROP ROLE` 语句用于从 MySQL 服务器中移除一个或多个角色。执行该操作需要具备 `CREATE USER` 或 `DROP ROLE` 权限（通常由管理员持有）。

删除角色后：

- 该角色被授予的所有权限将自动撤销。

- 之前被授予该角色的所有用户或角色将立即失去通过该角色获得的权限，但用户的直接权限（非通过角色授予的）不受影响。

- 已删除的角色名可以被重新创建，但不会自动关联回原来的授权关系。

```SQL
-- 删除名为 'manager' 的角色（默认主机为 %）
DROP ROLE 'manager';
-- 如果创建角色时指定了主机（如 'manager'@'localhost'），删除时也必须匹配
DROP ROLE 'manager'@'localhost';
```

#### 2\.4\.5激活角色

**激活角色的命令**

```SQL
-- 1. 在当前会话中激活指定角色
SET ROLE '角色名'[@'主机名'];
-- 2. 激活多个角色
SET ROLE '角色1', '角色2', '角色3';
-- 3. 激活当前用户的所有默认角色
SET ROLE DEFAULT;
-- 4. 激活用户被授予的所有角色
SET ROLE ALL;
-- 5. 排除某些角色后激活其余所有角色
SET ROLE ALL EXCEPT '角色1', '角色2';
```

```SQL
-- 激活单个角色
SET ROLE 'manager';
-- 激活默认角色（登录时自动激活的角色）
SET ROLE DEFAULT;
```

`SET ROLE` 语句用于在当前会话中激活一个或多个角色。角色被授予用户后，并不会自动生效，必须显式激活（或设置为默认角色）后，用户才能获得该角色所包含的权限。该命令仅在当前会话中有效，退出会话后失效。

激活角色后：

- 用户立即获得该角色被授予的所有权限（叠加用户自身的直接权限）。

- 可以通过 `SELECT CURRENT_ROLE();` 查看当前会话中处于激活状态的角色。

- 已激活的角色也可以通过 `SET ROLE NONE` 或 `SET ROLE DEFAULT` 切换为无角色或默认角色状态。

**设置默认角色（开机自动激活）**

为了避免每次登录都手动执行 `SET ROLE`，可以为用户指定默认角色：

```SQL
-- 为用户设置默认角色（登录后自动激活）
SET DEFAULT ROLE 'manager' TO '用户名'@'主机名';
-- 示例
SET DEFAULT ROLE 'manager' TO 'zhang'@'localhost';
```

设置默认角色后，用户登录时无需再执行 `SET ROLE`，角色的权限自动生效。

#### 2\.4\.6撤销用户的角色

撤销用户角色的SQL语法如下:

```SQL
REVOKE role FROM user;
```

练习:撤销oscar用户的manager角色。

在root用户下面执行:

```SQL
mysql> revoke manager from oscar;
Query OK, 0 rows affected (0.02 sec)
```

撤销后,执行如下查询语句,查看oscar用户的角色信息\(一定要退出oscar用户再重新登录\):

```SQL
SHOW GRANTS FOR 'oscar'@'%' ;-- 发现之前的角色己经不存在
```
